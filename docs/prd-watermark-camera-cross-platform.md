# UTS MarkVideo 水印相机 PRD

## 1. 目标

把 `uts-markvideo` 从“纯文字水印录视频 MVP”推进到“可配置水印的相机工具”：

- 水印支持文字和图片
- 水印样式可配置
- 水印可拖动定位，并支持缩放
- 拍照和录像都保留水印
- 录像时有系统相机风格的红点和计时
- 页面提供水印设置入口

当前实现阶段只做 iOS；本 PRD 同时保留给 Android 后续实现对齐。

> 说明：下面写的是目标规格，不是当前代码能力清单。当前只在 iOS 分支落地，Android 后续按同一份契约实现。

## 2. 范围总原则

- 先把“接口和语义”做成共有的，避免 iOS / Android 各自定义一套。
- 页面层统一负责水印设置、资源选择和结果展示，不再各自长出一套编辑器。
- 真正依赖系统相机、权限、相册、录制线程的部分，各平台自己实现。
- 本次开发只实现 iOS；Android 只保留 PRD 和共有契约，留给后续 Android 开发。
- 原生界面尽量保持 Android / iOS 统一，布局和层级要尽量同构。
- 原生界面要克制，避免过于复杂，减少两端样式偏差。
- 原生界面使用轻量阴影和层次感，但不做重装饰。

## 3. Shared

这部分是两端都要遵守的“公共约定”。

### 3.1 统一入口

- `recordWatermarkVideo(options)`
- `createWatermarkSample(options)` 继续保留，若后续不用也不在本 PRD 内删除

### 3.2 统一配置对象

`options.watermark` 由 Page 层整理后传给原生相机；原生层只消费已解析好的路径和样式，不负责请求 API。

图片水印遵循 Android 分支的契约：原生只接收 Page 已准备好的 `watermark.imagePath`，它必须是本地图片路径或可读取 URI。Page 层可以请求 `GET /api/watermark/logo-assets` 获取预设资源，接口返回 `logos[].imageUrl`、`name`、`width`、`height`；Page 负责把远程图片下载为本地临时路径，再写入 `watermark.imagePath`。如果接口不可用，Page 使用 `/static/watermark/company-logo.svg` 作为预设企业 Logo，用户也可以从相册替换。

| 字段 | 类型 / 默认值 | 语义 |
| --- | --- | --- |
| `text` | `string` / `''` | 文字水印；为空时使用原生默认文案；优先于旧版顶层 `text`。 |
| `imagePath` | `string` / `''` | 图片水印的本地路径或可读取 URI；由 Page 层把 API 资源解析后传入。 |
| `position` | `string` / `free` | 预留锚点字段；当前跨端以 `x/y` 为准，`free` 表示自由拖动。 |
| `x` / `y` | `number` / `0.5, 0.78` | 水印中心点在输出画面中的 0-1 归一化坐标；拖动后写回该值。 |
| `textColor` | `string` / `#FFFFFF` | 文字颜色，支持 `#RRGGBB`、`#RRGGBBAA`。 |
| `fontSize` | `number` / `30` | 文字大小，单位 px。 |
| `textBold` | `boolean` / `true` | 是否加粗。 |
| `imageWidth` / `imageHeight` | `number` / `72` | 图片水印尺寸，单位 px；只给一边时保持原始比例。 |
| `imageGap` | `number` / `18` | 图片与文字间距，单位 px。 |
| `boxWidth` / `boxHeight` | `number` / `0.88, 0.16` | 水印容器宽度和首选最小高度，按输出画面比例计算；图片 + 文字组合水印需要允许高度自适应，避免文字被裁切。 |
| `backgroundColor` | `string` / `#00000099` | 水印容器底色。 |
| `borderRadius` | `number` / `18` | 圆角，单位 px。 |
| `padding` | `number` / `28` | 内边距，单位 px。 |

水印以整块为单位进行拖动和缩放，不能只拖文字或只拖图标。

`options.camera`

| 字段 | 类型 / 默认值 | 语义 |
| --- | --- | --- |
| `facing` | `string` / `back` | `back` 或 `front`。 |
| `previewFit` | `string` / `cover` | `cover` 或 `contain`。 |
| `enablePhoto` | `boolean` / `false` | 显示拍照按钮；不单独切换成“纯拍照模式”。 |

开启 `enablePhoto` 后，用户可以在同一相机页内拍照；未录制时拍照返回 `kind: photo` 的结果。

`options.video`

| 字段 | 类型 / 默认值 | 语义 |
| --- | --- | --- |
| `width` / `height` | `number` / `0` | 仅保留兼容或样本用途；相机录制不作为用户可调项。 |
| `fps` | `number` / `30` | 目标帧率。 |
| `bitrate` | `number` / `2500000` | 目标码率，单位 bps。 |
| `includeAudio` | `boolean` / `true` | 是否录音。 |

`options.limits`

| 字段 | 类型 / 默认值 | 语义 |
| --- | --- | --- |
| `maxDurationMs` | `number` / `60000` | 最大录制时长，单位毫秒。 |
| `minDurationMs` | `number` / `1000` | 最小有效录制时长，单位毫秒。 |

`options.diagnostics`

| 字段 | 类型 / 默认值 | 语义 |
| --- | --- | --- |
| `perfLogging` | `boolean` / `false` | 仅用于开发调试。 |

### 3.3 统一结果对象

基础结果字段：

- `kind`: `recording` 或 `photo`
- `tempFilePath`: 主输出临时路径
- `savedFilePath`: 主输出保存引用
- `photoTempFilePaths`: 录制过程中额外拍到的照片临时路径列表
- `photoSavedFilePaths`: 录制过程中额外拍到的照片保存引用列表
- `durationMs`
- `width`
- `height`
- `watermarkText`: 实际烧录到画面里的文本

结果矩阵：

| kind | `tempFilePath` | `savedFilePath` | `photoTempFilePaths` | `photoSavedFilePaths` | `durationMs` |
| --- | --- | --- | --- | --- | --- |
| `recording` | 必填，MP4 临时路径 | 可选，系统相册保存引用 | 可选，若本次会话拍过照则返回 | 可选，若本次会话拍过照则返回 | 必填，实际录制时长 |
| `photo` | 必填，照片临时路径 | 可选，照片保存引用 | 空 | 空 | `0` |

`stats` 为可选诊断字段，Android 可以返回，iOS 可以省略。

`width` / `height` 始终描述实际输出分辨率；`recording` 取视频尺寸，`photo` 取照片尺寸。

### 3.4 统一交互语义

- `enablePhoto = true` 时显示拍照能力，不表示独立“纯拍照模式”
- 文字水印和图片水印允许同时显示
- 组合水印默认采用“上图标 / 下文字”的纵向布局
- 水印设置由独立页面承载，原生相机只消费保存后的配置；后续才考虑收敛为单界面弹框
- 水印支持以整个水印块为单位长按拖动，并通过双指捏合缩放
- 水印设置会记住上次配置并自动回填
- 录像中的红点和计时显示在顶部显眼位置
- 拍照和录像都保留当前水印
- 页面必须能区分“录像结果”和“仅照片结果”
- 水印设置的展示与编辑入口必须在页面上可见

## 4. Page

页面是两端都要共用的“用户入口层”。

### 4.1 必须有的控件

- 打开相机按钮
- 水印设置按钮（打开独立设置页）
- 拍照能力开关
- 文字水印输入
- 图片水印选择入口（预设资源由 API 获取，Page 负责下载、缓存和失败兜底）
- 水印样式输入区
- 结果展示区
- 水印设置页按文字 / 图片 / 样式 / 位置四块展开

### 4.2 页面状态

- 默认显示当前水印配置
- 默认回填上次保存的水印配置
- 默认回填已选图片资源和拍照能力状态
- 默认能看见是否开启拍照
- 录像时显示录制状态
- 拍照时显示拍照状态
- 结果区同时支持视频和照片

### 4.3 页面行为

- 保存当前水印配置后再进入原生相机
- 点击水印设置按钮进入独立设置页
- 设置页点保存后才生效
- 设置页提供恢复默认
- 拍照结果和录像结果回到页面后要能直接读懂
- 当开启拍照能力时，页面文案要明确拍照结果和录像结果是两种不同回传

## 5. iOS

本次只实现 iOS，以下是 iOS 本期范围。

### 5.1 相机界面

- 全屏相机预览
- 系统风格的拍照界面
- 水印预览
- 水印可长按拖动、可双指缩放
- 原生相机只消费 Page 已保存的配置，不内嵌水印编辑器
- 拍照按钮
- 录像按钮
- 完成按钮
- 整体样式尽量接近系统相机，不做复杂装饰
- 使用适度阴影增强层次，但不要破坏统一性

### 5.2 录像体验

- 录像时红点闪烁
- 录像时显示计时
- 录像状态位于顶部显眼位置
- 录像时保留水印
- 录像结束后返回视频结果

### 5.3 拍照体验

- 拍照时保留水印
- 拍照结果写入相册
- 拍照后可继续拍照或返回
- 拍照结果回传页面

### 5.4 权限

- 摄像头权限
- 麦克风权限
- 相册写入权限

## 6. Android

Android 先不实现代码，但 PRD 要先对齐后续方向。

### 6.1 后续要实现的东西

- 解析与 iOS 相同的公共契约
- 复用与 iOS 相同的页面语义
- 返回与 iOS 相同的结果结构
- 实现与 iOS 相近的拍照/录像交互、红点、计时和水印渲染
- 由 Page 层传入图片资源，不在原生层直接拉 API

### 6.2 Android 不要偏离的点

- 不要单独发明一套新的配置字段
- 不要把拍照结果设计成另一种返回格式
- 不要让页面在 Android / iOS 间出现不同的用户语义
- 不要把 Page 层独立设置页改成原生页内编辑器

## 8. 功能需求

### 8.1 水印配置

1. 支持输入文字水印。
2. 支持输入图片水印。
3. 支持文字和图片同时存在。
4. 支持设置文字样式和水印容器样式。
5. 支持在预览界面长按拖动整块水印。
6. 支持在预览界面通过双指捏合缩放整块水印。
7. 图片水印资源来自 API，下发的预设 icon 先用企业 logo 占位，并允许用户替换。

### 8.2 拍照

1. 进入相机后可拍照。
2. 拍照结果必须保留当前水印。
3. 拍照结果要写入系统相册。
4. 拍照结果要回传给页面。

### 8.3 录像

1. 进入相机后可录像。
2. 录像时显示红点闪烁。
3. 录像时显示当前已录制时间，格式为 `mm:ss`。
4. 录像结果必须保留当前水印。
5. 录像结果要回传给页面。

### 8.4 页面交互

1. 页面提供进入相机的按钮。
2. 页面提供水印设置按钮。
3. 页面提供拍照能力开关。
4. 页面能展示视频路径和照片保存引用。
5. 页面能区分“录像结果”和“仅照片结果”。

## 9. 体验要求

- 拍照界面尽量接近系统相机的直觉布局。
- 重要操作要一眼能找到。
- 录制中和拍照中，状态必须明确。
- 用户不能误把“结束录制”当成拍照完成按钮。
- 原生样式要尽量统一，避免 iOS / Android 出现明显层级差异。
- 原生界面保持克制，使用轻阴影即可，不做复杂装饰。

## 10. iOS 本期实现边界

- 先把相机交互、拍照、录像、相册保存、计时、红点做完整
- 先不做 Android 代码
